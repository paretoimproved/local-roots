`★ Insight ─────────────────────────────────────`
**Promise.all failure propagation**: The analytics page used `Promise.all([getAnalytics(), getPayoutHistory()])`. When the payouts endpoint 500'd due to missing DB columns, *both* results were lost — even though analytics worked fine. This is a common pattern worth noting: `Promise.allSettled` would have shown partial data (analytics metrics) while gracefully degrading on the failed endpoint. The current implementation treats the entire dashboard as all-or-nothing.

**Migration-gated features**: Phase 8 code shipped referencing columns (`transfer_error`, `transfer_retry_count`) from migration 0029, but migrations 0027-0031 weren't applied. In production, Railway would run migrations before the deploy — but locally, this is an easy gotcha after pulling new code.
